Curated resources for selecting a configuration management tool:

1. ["Getting Started with Configuration Management"](http://www.usenix.org/publications/login/2011-04/openpdfs/Lueninghoener.pdf) (USENIX ;login:, April 2011) — technical and social considerations
2. ["Configuration Management Summit"](http://www.usenix.org/publications/login/2010-10/openpdfs/ConfigMgt10reports.pdf) (USENIX ;login:, October 2010) — comparative overview of Bcfg2, CFEngine, Chef, and Puppet
3. ["A Survey of System Configuration Tools"](http://www.usenix.org/event/lisa10/tech/full_papers/Delaet.pdf) (LISA 2010) — framework for evaluating tools
4. [Comparison of Open Source Configuration Management Software](http://en.wikipedia.org/wiki/Comparison_of_open_source_configuration_management_software) — Wikipedia tabular summary
5. ["Puppet and CFEngine compared: time and resource consumption"](http://www.usenix.org/publications/login/2010-02/pdfs/bjorgeengen.pdf) (2010) — CFEngine's superior performance
6. [Why we use Cfengine: memory footprint](http://blog.normation.com/2011/02/23/why-we-use-cfengine-memory-footprint/)
7. [Why we use Cfengine: file editing](http://blog.normation.com/2011/03/21/why-we-use-cfengine-file-editing/)
8. [What Is Chef?](http://wiki.opscode.com/pages/viewpage.action?pageId=7274862) / [Why Chef?](http://wiki.opscode.com/pages/viewpage.action?pageId=7274864)
9. [A Visual Introduction to Chef](http://www.gliffy.com/publish/2886461/)
